ORIPA VAULT セットアップ

上から順番にやるだけ。コードはもう入っています。

1Supabase アカウントを作る

まだアカウントがなければ、ここから無料登録:

Supabase 無料登録

すでにログイン済みなら:

Supabase ダッシュボードを開く

2Supabase で「プロジェクト」を新規作成

  1. 下のボタンをクリック
  2. Organization を選ぶ（なければ New organization）
  3. Name: 好きな名前（例: oripa-vault）
  4. Database Password: 必ずメモ（後で使います）
  5. Region: Tokyo (Northeast Asia) を選ぶ
  6. Create new project → 1〜2分待つ
新規プロジェクト作成

3PC で初回セットアップを実行

エクスプローラーでこのフォルダを開く:

c:\Users\user1\projects\online-oripa

「初回セットアップ.bat」 をダブルクリック。

黒い画面が出て npm install が走ります。終わるまで待つ。

⚠ この時点では DB エラーが出ても OK。次のステップで .env.local を埋めたあと、もう一度実行します。

4Supabase の値をコピーして .env.local に貼る

4-1. 同じフォルダの 「設定リンクを開く.bat」 をダブルクリック

→ ブラウザに Supabase の設定ページが複数開き、メモ帳に .env.local が開きます。

4-2. 開いた Supabase ページから、メモ帳の該当行に貼り付け:

メモ帳の行Supabase でコピーする場所
NEXT_PUBLIC_SUPABASE_URLSettings → API → Project URL
NEXT_PUBLIC_SUPABASE_ANON_KEYSettings → API → anon public
SUPABASE_SERVICE_ROLE_KEYSettings → API → service_role（secret）
DATABASE_URLSettings → Database → URI → Transaction pooler (6543)
DIRECT_URLSettings → Database → URI → Session pooler または Direct (5432)
ADMIN_EMAIL管理者にしたい自分のメール（例: Gmail）

手動で開く場合（プロジェクト作成後）:

API Keys ページ Database 接続文字列

4-3. メモ帳を 上書き保存（Ctrl+S）して閉じる。

💡 [YOUR-PASSWORD] と書いてある部分は、ステップ2でメモした Database Password に置き換えてください。

5ログイン設定（Supabase で1回だけ）

5-1. Email ログインを ON

Auth → Providers を開く

→ Email を ON にする → Save

5-2. リダイレクト URL を追加

Auth → URL Configuration を開く

→ Redirect URLs に次を1行追加して Save:

http://localhost:3000/auth/callback

6もう一度「初回セットアップ.bat」を実行

.env.local を保存したあと、同じ 初回セットアップ.bat をもう一度ダブルクリック。

今度は DB（db:push）とサンプルデータ（seed）が入ります。

✅ 「セットアップ完了」と出れば OK

7アプリを起動

「起動.bat」 をダブルクリック。

ブラウザで次を開く:

http://localhost:3000（トップ） 会員登録 管理画面 ガチャ一覧

8管理者として登録

  1. 会員登録 を開く
  2. ADMIN_EMAIL に書いた同じメールで登録
  3. 管理画面 にアクセス → 管理者メニューが使える

困ったとき

症状やること
ログインできないEmail を ON + Redirect URL 確認
ガチャが空初回セットアップ.bat を再実行
DB エラー.env.local の DATABASE_URL / DIRECT_URL を確認
